# Design Systems y Componentes (design system, design tokens)

## 📜 Cita
> "A design system, which is a complete set of design standards, documentation and principles along with the toolkit (reusable components and patterns) can help design teams manage designs at scale... often considered a single source of truth." — Bornfight (2021)
> "The most important part of a design system is its documentation and a guide on using each component. Understanding not only the what, but the why, behind the design of a system is critical." — Marco Suarez (InVision)

## 📝 Conocimiento
- **Definición:** un **design system** es el conjunto completo de estándares, documentación y principios + un *toolkit* de componentes y patrones reutilizables; actúa como **single source of truth** para mantener coherencia en múltiples productos/plataformas.
- **Partes clave:** *component library*, *pattern library*, *style guide*, **design tokens**, *code snippets*, principios/branding.
- **Atomic Design (Brad Frost):** átomos → moléculas → organismos → templates → páginas; construye UI escalable desde piezas mínimas.
- **Design tokens:** variables (color, tipografía, espaciado, elevación) que centralizan decisiones de diseño y habilitan theming y escalado multiplataforma.
- **Beneficios:** consistencia de marca, escalabilidad, entrega más rápida (menos recrear), menor deuda de diseño; la documentación del *porqué* es tan importante como el componente.
- **Tooling:** Figma (design), Storybook (docs/catalog), integración con el código; conecta con `frontend.md` (React/Vue/Svelte) para componentes reales.
- **Relevancia MVAS:** cruza con `accesibilidad.md` (WCAG en componentes), `ux-research.md`, `design-thinking.md`, `service-design.md` y los roles `rol/design-lead`, `rol/ux-ui-lead`; base de `ux-writing.md` (los componentes incluyen microcopy).
