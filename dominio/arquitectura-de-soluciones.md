# Arquitectura de soluciones (Solution Architecture)

## Definición
Disciplina que describe, diseña y gestiona la **solución técnica** a un problema de negocio: cómo componentes de software/hardware, datos y procesos se combinan para cumplir requisitos funcionales y no funcionales. Es el puente entre la estrategia empresarial y la ejecución técnica.

## Responsabilidades del arquitecto de soluciones
- Traducir requisitos de negocio a diseño técnico.
- Seleccionar patrones (microservicios, event-driven, CQRS, API gateway) y tecnologías.
- Evaluar **trade-offs** (costo/escalabilidad/mantenibilidad/seguridad).
- Definir ADRs (Architecture Decision Records), diagramas (C4, UML), RFPs.
- Colaborar con `rol/solutions-architect` (rol) y `dominio/arquitectura-empresarial`.

## Marcos de referencia
TOGAF (ADM), **SARA** (Solution Architecture Reference Architecture), Zachman. Diferente de enterprise architecture (más amplio, alineación org) y de `dominio/estilos-arquitectura` (patrones puros).

## Entregables
Documento de arquitectura de solución, diagramas de componentes/despliegue, matriz de requisitos, plan de migración. Crítico en `dominio/consultoria`, `dominio/transformacion-digital` y proyectos de `dominio/cloud-computing`.
