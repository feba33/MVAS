# Ingeniería de software

La **ingeniería de software** aplica principios de ingeniería al desarrollo, operación y mantenimiento de software robusto y escalable. Abarca el ciclo de vida completo, desde requisitos hasta operación (SRE/DevOps).

## Ciclo de vida (SDLC)
- **Fases:** requisitos → diseño → implementación → pruebas → despliegue → operación/mantenimiento.
- **Modelos:** cascada, iterativo, ágil (Scrum/Kanban), spiral, DevOps/DevSecOps continuo.
- **Requisitos y arquitectura:** patrones (MVC, microservicios, event-driven), calidad de diseño.

## Prácticas clave
- **Control de versiones** (Git), integración continua/despliegue continuo (CI/CD).
- **Calidad:** pruebas unitarias/integración/e2e, code review, static analysis.
- **Observabilidad:** logging, métricas, tracing.
- **Seguridad:** DevSecOps, SAST/DAST, threat modeling. vs `dominio/ciberseguridad-de-aplicaciones`.
- **Entrega:** infraestructura como código (IaC), containers (Docker/K8s).

## Disciplinas relacionadas
- vs `dominio/arquitectura-de-soluciones` (patrones de arquitectura), `dominio/devops` (prácticas de entrega), `dominio/desarrollo-de-software` (si existe), `dominio/calidad-de-software` (QA).
- Roles: software engineer, SRE, DevOps engineer, QA engineer, arquitecto de software.

## Estándares
- ISO/IEC 12207 (ciclo de vida), IEEE 830 (requisitos), SWEBOK (cuerpo de conocimiento).
