# Nolvorn (organización)

Nolvorn es una firma de software & technology partners (fundada 2026, HQ Tampico,
Tamaulipas, México). Esta carpeta es la **entidad canónica de Nolvorn dentro del
super-wiki MVAS**, sincronizada con dos fuentes vivas:

- **Plane.so** (self-hosted en `plane.mvaslabs.cloud`, workspace `nolvorn`,
  proyecto `General` / identifier `NOLVO`) — sistema de tickets / acción.
- **Google Drive Shared Drive "nolvorn"** (`0ADXc9xJB9XxQUk9PVA`) — knowledge base
  operativa, organizada con el sistema **PARA** (Projects / Areas / Resources / Archive).

## Principio de bucle bidireccional (Drive ↔ MVAS)
Se busca un ciclo de mejora continua:
1. **Ingesta (Drive → MVAS):** el contenido del Drive (sobre todo
   `Org/`, `Resources/`, `README`, `nolvorn_master_context_v1`) se refleja y
   sintetiza en las carpetas de esta entidad.
2. **Optimización suave (MVAS → Drive):** Hermes propone mejoras al Drive, pero
   **NUNCA modifica el Drive sin autorización explícita previa**. Escala planes y
   necesidades y pide aprobación antes de cualquier write/share/delete.
3. El `master_context` del Drive es la fuente de identidad; MVAS lo enriquece con
   decisiones, protocolos y responsabilidades que evolucionan.

## Subcarpetas
- `responsabilidades/` — roles y deberes (incl. los de Hermes en este chat).
- `protocolos/` — procesos y formas de operar (ingest, comunicación, autorización).
- `decisiones/` — cronología de decisiones tomadas en Nolvorn.
- `conocimiento/` — síntesis y notas transversales (lo que Hermes deduce/organiza).
- `kb-drive/` — espejo estructurado del Shared Drive (índice de carpetas/doc IDs).

## Convención de estilo (heredada del Drive)
Todo documento debe ser **plain, minimal formatting** — headings y estructura solo
para legibilidad, sin color ni decoración. Aplica en MVAS y en propuestas al Drive.
