# Software Engineer

## 📝 Conocimiento
- **Rol:** rama de la ciencia de la computación y la ingeniería enfocada en diseñar, desarrollar, probar y mantener software; aplica principios de ingeniería y programación para satisfacer necesidades del usuario.
- **Ciclo (SDLC):** define, implementa, prueba, gestiona y mantiene sistemas de software mediante requisitos → diseño → construcción → pruebas → despliegue → operación.
- **Prácticas:** clean code, testing automatizado (unit/integration/e2e), code review, CI/CD, metodologías ágiles (Scrum/Kanban) y arquitectura (monolito, microservicios, event-driven).
- **Niveles:** Junior → Mid → Senior → Staff/Principal (ver `rol/staff-engineer`); el track IC vs management (ver `rol/engineering-manager`).
- **Ecosistema:** solapa con `rol/devops-engineer`, `rol/platform-engineer`, `rol/data-engineer` y `rol/ml-engineer`; base de `dominio/ingenieria-software`.
- **Relevancia:** conecta con `dominio/arquitectura-software`, `dominio/metodologias-agiles`, `rol/engineering-manager` y `rol/staff-engineer`.
