# DevSecOps Engineer

## 📝 Conocimiento
- **Rol:** integra la seguridad en el ciclo de vida de software (plan → develop → build → test → release → deploy → operate) como código y automatización, no como gate manual.
- **Prácticas:** shift-left (seguridad desde el diseño), security as code, SAST/DAST/SCA/IAST, gestión de secretos, hardening de imágenes, políticas en CI/CD, supply chain (SBOM, SLSA, firma), IaC scanning, threat modeling.
- **Tooling:** CI/CD (GitHub Actions, GitLab, Jenkins), registries, scanners (Semgrep, Trivy, Snyk), CSPM, vaults.
- **Diferenciación:** vs `rol/devops-engineer` (velocidad/operación, seguridad como responsabilidad compartida) y `rol/security-engineer` (seguridad general/blue team) — DevSecOps lleva la seguridad al pipeline.
- **Relevancia:** conecta con `rol/devops-engineer`, `rol/security-engineer`, `dominio/plataforma-sre`, `rol/ciso`, `dominio/ciberseguridad`.
