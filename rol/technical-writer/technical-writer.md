# Technical Writer

## 📝 Conocimiento
- **Rol:** produce **documentación técnica** que permite a usuarios y desarrolladores usar, integrar y operar productos y sistemas: manuales, guías de usuario, **API docs**, runbooks, READMEs, guías de arquitectura.
- **Docs-as-code:** authoring en **Markdown/AsciiDoc dentro de git**, generación con static site generators (Docusaurus, Sphinx, MkDocs), revisiones por PR, CI para validar links/ builds. Integra documentación al flujo de ingeniería.
- **Públicos:** desarrolladores (reference/API), usuarios finales (how-to), internos (runbooks, onboarding), soporte/ventas (enablement).
- **Herramientas:** Confluence, Notion, Docusaurus, Swagger/OpenAPI, MadCap, Git. Métricas: findability, tareas completadas, feedback, tiempo de localización.
- **Habilidades:** escritura clara y estructurada, pensamiento analítico, empatía de usuario, capacidad técnica para leer código/diagramas, gestión de información.
- **Diferenciación:** vs `dominio/diseno/ux-writing` (microcopy de producto), vs `rol/community-manager` (docs comunitarias), vs marketing/content. Reporta a Product/Eng/Documentation.
- **Relevancia:** cruza producto, ingeniería y soporte. Conecta con `dominio/computacion`, `dominio/diseno/ux-writing`, `dominio/experiencia-cliente` y `dominio/producto`.
