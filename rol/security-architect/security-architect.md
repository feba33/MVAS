# Security Architect

## 📝 Conocimiento
- **Qué es:** rol de ciberseguridad que **diseña** la arquitectura de seguridad de sistemas, aplicaciones y datos a nivel de organización. Se sitúa entre el `rol/security-engineer` (quien implementa) y el `rol/ciso` (quien fija estrategia ejecutiva).
- **Responsabilidades:** *threat modeling* (STRIDE, PASTA), definir controles por capa (red, identidad, aplicación, datos), *reference architectures*, adopción de **Zero Trust**, selección de estándares (NIST CSF, ISO 27001, CIS Controls), revisión de diseños y *secure-by-design*.
- **Entregables:** diagramas de arquitectura de seguridad, políticas, patrons de control, guías de implementación, participación en arquitectura de soluciones.
- **Habilidades:** seguridad ofensiva/defensiva, nubes (AWS/Azure/GCP), IAM, criptografía, redes, y capacidad de traducir riesgo de negocio a controles técnicos.
- **vs** `rol/security-engineer`, `rol/ciso`, `rol/devsecops-engineer`, `dominio/ciberseguridad`, `dominio/ciberdefensa`.
- **Relevancia MVAS:** define el control de `dominio/proteccion-de-datos`, `dominio/gobierno-de-ti`, `sustrato/estados-unidos/seguridad-nacional`.
